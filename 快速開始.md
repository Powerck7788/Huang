# 快速開始指南

## 🚀 一分鐘快速啟動

這是最快速的方式來開始使用性能優化工具包！

### 第一步：下載

```bash
git clone https://github.com/Powerck7788/Huang.git
cd Huang
```

### 第二步：安裝

```bash
./安裝.sh
```

### 第三步：使用

安裝完成後，您就可以開始使用了！

```bash
# 分析您的應用程式
npm run build:analyze

# 檢查性能
npm run lighthouse

# 檢查文件大小
npm run size-limit
```

---

## 📖 詳細說明

### 安裝腳本會做什麼？

執行 `./安裝.sh` 後，腳本會：

1. ✅ 檢查您的系統是否有 Node.js 和 npm
2. ✅ 詢問您要安裝什麼工具（推薦選擇「完整安裝」）
3. ✅ 自動安裝所有需要的 npm 套件
4. ✅ 創建配置文件（webpack、lighthouse、size-limit 等）
5. ✅ 設置 npm 腳本，方便您日後使用
6. ✅ 創建示例代碼和工具函數
7. ✅ 設置 GitHub Actions（可選）

### 安裝選項

安裝時會讓您選擇：

| 選項 | 說明 | 適合 |
|------|------|------|
| **1. 完整安裝** | 安裝所有工具和功能 | 🌟 推薦給所有用戶 |
| **2. 基本安裝** | 只安裝核心性能監控工具 | 輕量級專案 |
| **3. 自定義安裝** | 自己選擇要安裝什麼 | 有特定需求的用戶 |

## 🎯 安裝後可以做什麼？

### 1. 監控網頁性能

在您的應用程式中加入：

```javascript
import { initPerformanceMonitoring } from './src/utils/performance.js';

// 啟動性能監控
initPerformanceMonitoring();
```

這會自動追蹤：
- ⏱️ 首次內容繪製 (FCP)
- 🖼️ 最大內容繪製 (LCP)
- 🖱️ 首次輸入延遲 (FID)
- 📐 累積版面配置位移 (CLS)
- 🌐 首次位元組時間 (TTFB)

### 2. 分析 Bundle 大小

```bash
npm run build:analyze
```

會產生一個互動式的圖表，顯示：
- 📊 每個模組的大小
- 📦 哪些套件佔用最多空間
- 🔍 可以優化的地方

### 3. 執行 Lighthouse 審查

```bash
# 先啟動您的應用程式
npm start

# 在另一個終端機執行
npm run lighthouse
```

Lighthouse 會檢查：
- 🚀 性能
- ♿ 可訪問性
- 🎯 最佳實踐
- 🔍 SEO

### 4. 設置文件大小限制

編輯 `.size-limit.json` 設置您的目標：

```json
[
  {
    "path": "dist/**/*.js",
    "limit": "200 KB"  // 修改為您的目標
  },
  {
    "path": "dist/**/*.css",
    "limit": "50 KB"   // 修改為您的目標
  }
]
```

然後檢查：

```bash
npm run size-limit
```

## 💡 實用範例

### 範例 1：測量功能執行時間

```javascript
import { performanceMark, performanceMeasure } from './src/utils/performance.js';

function myFeature() {
  performanceMark('feature-start');
  
  // 您的代碼...
  doSomething();
  
  performanceMark('feature-end');
  performanceMeasure('feature-duration', 'feature-start', 'feature-end');
}
```

### 範例 2：分析資源載入

```javascript
import { analyzeResourceTiming } from './src/utils/performance.js';

// 在頁面載入完成後
window.addEventListener('load', () => {
  const analysis = analyzeResourceTiming();
  console.log('總資源數:', analysis.totalResources);
  console.log('總大小:', analysis.totalSize);
  console.log('慢速資源:', analysis.slowestResources);
});
```

### 範例 3：與 React 整合

```javascript
import React, { useEffect } from 'react';
import { initPerformanceMonitoring } from './utils/performance';

function App() {
  useEffect(() => {
    // 在應用程式載入時初始化性能監控
    initPerformanceMonitoring();
  }, []);

  return (
    <div className="App">
      {/* 您的應用程式 */}
    </div>
  );
}

export default App;
```

### 範例 4：與 Vue 整合

```javascript
import { createApp } from 'vue';
import App from './App.vue';
import { initPerformanceMonitoring } from './utils/performance';

const app = createApp(App);

// 初始化性能監控
initPerformanceMonitoring();

app.mount('#app');
```

## 🔍 常見使用情境

### 情境 1：新專案開始

```bash

cd my-app

# 2. 複製安裝腳本

chmod +x 安裝.sh

# 3. 執行安裝

# 4. 開始開發
npm start
```

